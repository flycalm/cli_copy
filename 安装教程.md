# Remote Terminal Image Paste 安装教程

这份教程只讲最终保留下来的正式方案：

- 唯一方案：`remoteFs`
- 默认快捷键：`Ctrl+Alt+V`
- 快捷键可在插件设置中切换

这个扩展的目标场景是：

- 本地电脑是 Windows
- 本地运行 VS Code
- 通过 Remote-SSH 连接 Ubuntu 开发机
- 远端终端里跑 `codex`、`claude code`、`gemini` 等 CLI
- 你在 Windows 上截图后，希望直接把图片以“远端文件 + 路径引用”的方式塞进终端

## 方案说明

这个版本不再保留其他实验路线，只保留已经验证可行的 `remoteFs`：

1. 本地 VS Code 扩展读取 Windows 剪贴板里的图片
2. 通过 `vscode.workspace.fs` 把图片写到远端工作区
3. 再把远端路径插入当前终端

这条路线最适合你的真实使用场景。

## 方式一：开发模式直接安装使用

这是你当前最适合的方式。

### 第 1 步：打开你本地 Windows 上的项目目录

先确认项目在你本地 Windows 上的实际位置。

例如可能是：

- `D:\dev\cli_copy`
- `C:\Users\你的用户名\Desktop\cli_copy`

然后用 VS Code 打开那个目录。

这个目录里应该能看到：

- `package.json`
- `src`
- `.vscode`

### 第 2 步：安装依赖

在 VS Code 终端里执行：

```powershell
npm install
```

执行完成后再运行：

```powershell
npm run build
```

### 第 3 步：启动扩展开发宿主

在 VS Code 里按 `F5`。

正常情况下会打开一个新的窗口，名字通常是：

`Extension Development Host`

以后所有测试都在这个新窗口里做。

如果你按 `F5` 时弹出“选择调试器”，说明当前目录没有被当成扩展工程正确打开，或者 VS Code 还没读取调试配置。

这个项目已经带了调试配置：

- [launch.json](/home/cc/mywork/cli_copy/.vscode/launch.json):1
- [tasks.json](/home/cc/mywork/cli_copy/.vscode/tasks.json):1

### 第 4 步：在开发宿主里连 Remote-SSH

在 `Extension Development Host` 这个新窗口里：

1. 安装官方 `Remote - SSH`
2. 使用 `Remote-SSH: Connect to Host...`
3. 连到你的 Ubuntu 开发机
4. 打开远端工作区
5. 打开远端集成终端

### 第 5 步：测试图片粘贴

在 Windows 本地先截图，让图片进入系统剪贴板。

然后在 `Extension Development Host` 窗口里：

1. 聚焦远端终端
2. 按 `Ctrl+Alt+V`

默认行为：

- 扩展读取 Windows 剪贴板图片
- 把图片写到远端 `.vscode-ai-images`
- 把路径插入终端输入框

如果剪贴板里不是图片，就自动回退成普通终端粘贴。

## 方式二：打包成 VSIX 再安装

如果你想像普通 VS Code 插件那样安装，可以走这个方式。

### 第 1 步：安装打包工具

```powershell
npm install -g @vscode/vsce
```

如果不想全局安装，也可以直接用 `npx`。

### 第 2 步：打包

在你本地 Windows 的项目目录里执行：

```powershell
npm install
npm run build
npx @vscode/vsce package
```

执行成功后，项目目录里会出现一个 `.vsix` 文件。

### 第 3 步：安装 VSIX

在本地 Windows 的 VS Code 里：

1. 打开扩展侧边栏
2. 右上角点 `...`
3. 选择 `Install from VSIX...`
4. 选择刚才生成的 `.vsix`

安装后重启 VS Code。

## 插件设置

### 快捷键设置

默认是：

```json
{
  "remoteTerminalImagePaste.shortcut": "ctrlAltV"
}
```

可选值：

- `ctrlShiftV`
- `ctrlAltV`
- `altV`
- `none`

如果你设成 `none`，就不注册快捷键，只通过命令面板调用。

### Claude Code 推荐设置

如果你希望插入的是 Claude 风格的文件引用，建议这样配：

```json
{
  "remoteTerminalImagePaste.shortcut": "ctrlAltV",
  "remoteTerminalImagePaste.insertTemplate": "@{path}"
}
```

### 通用路径插入

```json
{
  "remoteTerminalImagePaste.shortcut": "ctrlAltV",
  "remoteTerminalImagePaste.insertTemplate": "{path}"
}
```

### 远端目录

```json
{
  "remoteTerminalImagePaste.remoteDirectory": ".vscode-ai-images"
}
```

## 命令面板命令

在命令面板，也就是 `Ctrl+Shift+P` 里，你可以执行：

- `Remote Terminal Image Paste: Paste Image To Remote Terminal`
- `Remote Terminal Image Paste: Cleanup Saved Images`

注意：

- 这些是 VS Code 命令
- 不是远端 Ubuntu shell 命令
- 不要在 `cc@...$` 这种终端提示符里直接输入

## 推荐测试方式

我建议按这个顺序测试：

1. 先在 `Extension Development Host` 里连上 Remote-SSH
2. 在远端终端启动 `codex` 或 `claude`
3. Windows 本地截图
4. 先按默认快捷键 `Ctrl+Alt+V`
5. 如果你想进一步避开冲突，就把插件设置里的快捷键改成 `Alt+V`

## 常见问题

### 1. 为什么不是直接把图片二进制粘贴到终端里？

因为 VS Code 集成终端和 Remote-SSH 的输入链路本质上是文本流，不是图片对象通道。

所以最稳的做法是：

- 先把图片写到远端
- 再把远端路径输入到终端

### 2. 为什么这个插件必须在本地 Windows 的 VS Code 里运行？

因为它要读取的是 Windows 本地系统剪贴板。

本项目已经声明自己是 UI 扩展：

- [package.json](/home/cc/mywork/cli_copy/package.json):34

### 3. 如果快捷键没有触发怎么办？

先做这几步：

1. 确认你在 `Extension Development Host` 窗口里
2. 确认焦点在终端
3. 打开 VS Code 设置，检查 `remoteTerminalImagePaste.shortcut`
4. 如果当前快捷键冲突，就把它改成 `altV`
5. 或者直接用命令面板执行 `Remote Terminal Image Paste: Paste Image To Remote Terminal`

### 4. 如果我只想清理远端图片怎么办？

执行命令面板命令：

- `Remote Terminal Image Paste: Cleanup Saved Images`
